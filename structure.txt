

# 🧠 🔥 MASTER PATTERN (Service Function Template)

Whenever you write any service function, think in **5 steps**:

```ts
async function actionName(input?) {
  try {
    // 1. Get cookies (auth)
    const cookieStore = await cookies()

    // 2. Call API (method + URL)
    const res = await fetch(URL, {
      method: "...",
      headers: {
        "Content-Type": "application/json", // only if needed
        Cookie: cookieStore.toString(),
      },
      body: JSON.stringify(input), // only for POST/PATCH
      next: { tags: ["tag-name"] }, // only for GET caching
    })

    // 3. Parse response
    const data = await res.json()

    // 4. Return success
    return { data, error: null }

  } catch (error) {
    // 5. Return error
    return { data: null, error: { message: "...", details: error } }
  }
}
```

# 🔥 Now simplify into MEMORY RULES

## 🟢 RULE 1: Always start with cookies

```ts
const cookieStore = await cookies()
```

👉 Because:

* Every protected route needs auth

---

## 🟢 RULE 2: Choose HTTP method

| Operation | Method |
| --------- | ------ |
| Create    | POST   |
| Get       | GET    |
| Update    | PATCH  |
| Delete    | DELETE |

---

## 🟢 RULE 3: Body only for POST/PATCH

```ts
body: JSON.stringify(data)
```

❌ No body in GET
❌ No body in DELETE (usually)

---

## 🟢 RULE 4: Content-Type only when sending body

```ts
"Content-Type": "application/json"
```

---

## 🟢 RULE 5: Always send cookies

```ts
Cookie: cookieStore.toString()
```

---

## 🟢 RULE 6: Cache only for GET

```ts
next: { tags: ["something"] }
```

# 🧩 🔥 OPERATION-WISE STRUCTURE (Easy to Remember)

---

## ✅ 1. CREATE (POST)

```ts
createSomething: async (input) => {
  try {
    const cookieStore = await cookies()

    const res = await fetch(`/endpoint`, {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        Cookie: cookieStore.toString(),
      },
      body: JSON.stringify(input),
    })

    const data = await res.json()
    return { data, error: null }

  } catch (error) {
    return { data: null, error: { message: "Create failed", details: error } }
  }
}
```

🧠 Think: **Send data → create**

---

## ✅ 2. GET (LIST / SINGLE)

```ts
getSomething: async () => {
  try {
    const cookieStore = await cookies()

    const res = await fetch(`/endpoint`, {
      headers: { Cookie: cookieStore.toString() },
      next: { tags: ["tag-name"] },
    })

    const data = await res.json()
    return { data, error: null }

  } catch (error) {
    return { data: null, error: { message: "Fetch failed", details: error } }
  }
}
```

🧠 Think: **Just read → no body**

---

## ✅ 3. GET BY ID

```ts
getById: async (id: string) => {
  try {
    const cookieStore = await cookies()

    const res = await fetch(`/endpoint/${id}`, {
      headers: { Cookie: cookieStore.toString() },
      next: { tags: [`item-${id}`] },
    })

    const data = await res.json()
    return { data, error: null }

  } catch (error) {
    return { data: null, error: { message: "Fetch failed", details: error } }
  }
}
```

🧠 Think: **Dynamic URL**

---

## ✅ 4. UPDATE (PATCH)

```ts
updateSomething: async (id, input) => {
  try {
    const cookieStore = await cookies()

    const res = await fetch(`/endpoint/${id}`, {
      method: "PATCH",
      headers: {
        "Content-Type": "application/json",
        Cookie: cookieStore.toString(),
      },
      body: JSON.stringify(input),
    })

    const data = await res.json()
    return { data, error: null }

  } catch (error) {
    return { data: null, error: { message: "Update failed", details: error } }
  }
}
```

🧠 Think: **Update existing data**

---

## ✅ 5. DELETE

```ts
deleteSomething: async (id: string) => {
  try {
    const cookieStore = await cookies()

    const res = await fetch(`/endpoint/${id}`, {
      method: "DELETE",
      headers: { Cookie: cookieStore.toString() },
    })

    const data = await res.json()
    return { data, error: null }

  } catch (error) {
    return { data: null, error: { message: "Delete failed", details: error } }
  }
}
```

🧠 Think: **Remove → no body**
